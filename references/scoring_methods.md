# 评分方法

## 1. 维度分评分原则

每个维度按 1-10 分打分：
- 1-3：很差 / 远低于平均
- 4-6：一般 / 平均水平
- 7-8：良好 / 优于平均
- 9-10：优秀 / 顶尖

## 2. 价格维度评分公式

价格越低，分数越高：
`价格分 = min(10, 10 × (最低价格 / 该方案价格))`，结果四舍五入保留一位小数。

## 3. 加权总分计算

```
总分 = Σ (维度分_i × 权重_i)
```
权重和为 1（即 100%），总分范围 1-10。

## 4. 信息缺失处理

如果某方案在某维度的信息缺失：
- 优先追问用户补充
- 若用户无法提供，则：
  - 如果其他方案都有该维度信息 → 缺失方此项按 3 分计入，并标注"信息缺失，暂按较低分处理"
  - 如果多数方案都缺失 → 该维度不参与评分，权重重新分配给其他维度

## 5. 用户偏好到权重的映射（自然语言解析）

| 用户说法 | 权重调整 |
|---------|---------|
| "我最看重价格 / 性价比" | 成本权重提升至 50%，其他各降 |
| "我要风险最低" | 风险权重提升至 40% |
| "预算不超过 X 元" | 价格 > X 的方案自动排除，不参与评分 |
| "我要功能最强的，钱不是问题" | 核心价值权重提升至 60% |
| "我就要省心" | 可行性权重提升至 40% |

组合偏好：按比例分配，确保总和为 100%。

## 6. 推荐强度判定

- 最高分与第二名分差 ≥ 1.5 分 → "✅ 强烈推荐"
- 分差 0.5-1.5 分 → "✅ 推荐"
